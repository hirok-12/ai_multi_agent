# AI Multi-Agent (AIブログ自動化システム)

このプロジェクトは、最新のAIトレンドの調査からSEOに最適化されたブログ記事の作成、そしてNotionへの投稿までを自動化するマルチエージェントシステムです。

## 概要

以下の2つのエージェントが連携してタスクを遂行します。

1. **marketing (マーケティングエージェント)**
   - 最新のAIニュースやトレンドをウェブ検索で調査します。
   - 話題性・新規性・実用性の観点からバズりやすいトピックを評価し、上位3件を選定します。

2. **content-leader (コンテンツリーダーエージェント)**
   - マーケティングエージェントからの調査結果をもとに、SEOに配慮したブログ記事（2,000〜3,000字）を執筆します。
   - 執筆終了後、指定されたNotionのページまたはデータベースに自動で記事を投稿します。

## 使い方

Claude（AIアシスタント）に対して、以下のようなプロンプトを入力して処理を開始します。

**実行プロンプトの例:**
> 「@marketing で最新のAIトレンドを調査し、バズりそうなトピックを教えてください。その後、@content-leader でその結果をもとにSEOブログ記事を書き、Notionに投稿してください。」

> 「今週のAIニュースについて調べて、ブログ記事にしてNotionにアップして。」

アシスタントがリクエストを解釈し、適切なエージェント（marketing → content-leader の順）を呼び出して処理を自動進行します。

## 動作要件 (前提条件)

- **Notion連携 (MCP)**: `content-leader` がNotionにアクセスするため、Notion用MCPサーバー（`notion-search`, `notion-fetch`, `notion-create-pages`, `notion-update-page`）が正しく設定され、投稿先へのアクセス権限が付与されている必要があります。
- **Web検索**: `marketing` が情報を取得するために `WebSearch` ツールが有効化されている必要があります。

## プロジェクトのルール
- すべての出力は**日本語**で行われます。
- ターゲット層は「AIに興味のある非エンジニア」です。
- 記事は事実に基づき、話題性と検索意図（SEO）を満たす内容に調整されます。
